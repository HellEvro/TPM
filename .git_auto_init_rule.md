# Automatic Git Repository Initialization Rule

## CRITICALLY IMPORTANT: Auto-initialize Git Repository

**MANDATORY behavior for ALL projects:**

### When to initialize Git:
- When creating a new project from scratch
- When working in a project directory that does not have a `.git` folder
- When `git status` command fails with "not a git repository" error

### Automatic Git initialization process:

1. **Check for existing Git repository:**
   - Execute `git status` or check if `.git` directory exists
   - If Git is already initialized, skip initialization

2. **Initialize Git repository if missing:**
   - Execute `git init` in the project root directory
   - Do NOT create initial commit automatically (let user decide when to make first commit)

3. **Create basic .gitignore if missing:**
   - If `.gitignore` file does not exist, create a basic one with common patterns:
     - `__pycache__/`
     - `*.pyc`
     - `*.pyo`
     - `*.pyd`
     - `.Python`
     - `env/`
     - `venv/`
     - `.venv`
     - `.env`
     - `.DS_Store`
     - `*.log`
     - `.idea/`
     - `.vscode/`
     - `node_modules/`
   - If `.gitignore` already exists, do NOT overwrite it

4. **When to perform check:**
   - Before first commit attempt
   - When user asks to commit changes
   - When working in a new project directory
   - Proactively check when starting work on a project

### Important notes:
- Always check for Git before attempting any git commands
- Never force initialize if user explicitly doesn't want Git
- If initialization fails, inform user but continue with other tasks
- Respect existing Git configuration and do not modify it
