# Исправление спама в логах синхронизации позиций

## Проблема
Система выдавала предупреждения "Не удалось получить позиции с биржи - пропускаем синхронизацию" даже когда на бирже просто нет активных позиций. Это создавало спам в логах.

## Причина
Функция `get_exchange_positions()` возвращала пустой список `[]` как при отсутствии позиций (нормально), так и при ошибках API (проблема). Система проверяла `if not exchange_positions:` что срабатывало в обоих случаях.

## Исправления

### 1. Изменена логика проверки в `sync_positions_with_exchange()`
```python
# БЫЛО:
if not exchange_positions:
    logger.warning("[POSITION_SYNC] ⚠️ Не удалось получить позиции с биржи - пропускаем синхронизацию")
    return False

# СТАЛО:
if exchange_positions is None:
    logger.warning("[POSITION_SYNC] ⚠️ Не удалось получить позиции с биржи - пропускаем синхронизацию")
    return False
```

### 2. Изменена логика проверки в `cleanup_inactive_bots()`
```python
# БЫЛО:
if not exchange_positions:
    logger.warning(f"[INACTIVE_CLEANUP] ⚠️ Не удалось получить позиции с биржи - пропускаем очистку для безопасности")
    return False

# СТАЛО:
if exchange_positions is None:
    logger.warning(f"[INACTIVE_CLEANUP] ⚠️ Не удалось получить позиции с биржи - пропускаем очистку для безопасности")
    return False
```

### 3. Исправлена функция `get_exchange_positions()`
Теперь функция возвращает:
- `[]` - когда нет позиций (нормально)
- `None` - при ошибках API или неинициализированной бирже

```python
# Исправлены все места где возвращался [] при ошибках:
return None  # вместо return []
```

## Результат
- ✅ Убран спам в логах когда нет позиций
- ✅ Предупреждения показываются только при реальных ошибках
- ✅ Система корректно различает "нет позиций" и "ошибка получения позиций"
